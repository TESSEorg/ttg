Documentation

A device::Task is a coroutine (a callable that can be suspended and resumed).

Since task execution in TTG is not preempable, tasks should not block. The purpose of suspending a task is to yield control back to the runtime until some events occur; in the meantime its executor (e.g., a user-space thread) can perform other work. Once the task function reaches a point where further progress is pending completion of one or more asynchronous actions the function needs to be suspended via a coroutine await (co_await). Resumption will be handled by the runtime.
